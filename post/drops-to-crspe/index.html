部署

环境准备(m.s)

JAVA部署

关闭防火墙

systemctl stop firewalld
systemctl disable firewalld

关闭selinux：

vi /etc/sysconfig/selinux
SELINUX=disabled

安装java

解压至指定位置

tar -zxvf filename

配置变量

vi /etc/profile

#java
export JAVA_HOME='yourjavapath'
export PATH=$JAVA_HOME/bin:$PATH

保存退出后，执行：

source /etc/profile

查看版本，确定环境变量是否生效

java -version

配置ssh

安装ssh

yum install ssh

启动ssh

service sshd start

查看ssh进程

systemctl status sshd
配置ssh免密登录

namenode生成密钥：

ssh-keygen -t dsa -P '' -f ~/.ssh/id_dsa

ssh-keygen

此时在~/.ssh/目录下生成id_dsa.pub(/root/.ssh/id_rsa)文件

若希望master节点无需密码即可ssh连接到slave节点，需将master结点上的/.ssh/id_dsa.pub的内容，添加到slave节点/.ssh/authorized_keys中。

若slave节点/.ssh/authorized_keys不存在，直接将/.ssh/id_dsa.pub文件复制过去并重命名为authorized_keys：

scp ~/.ssh/id_dsa.pub username@slavenode_ip:~/.ssh/authorized_keys

若提示scp: /root/.ssh/authorized_keys: No such file or directory(~/.ssh)

slave节点上:

mkdir ~/.ssh/

配置hosts

修改hostname

vi /etc/hostname

修改hosts（匹配ip与hostname映射关系）

vi /etc/hosts

例：

10.1.1.11	master
10.1.1.21	slave-1
10.1.1.22	slave-2

组件部署

1. Hadoop部署

安装

master中

安装包解压到指定位置

tar -xzvf hadooppackage.tar.gz -C location

新建备用目录

mkdir  /root/hadoop
mkdir  /root/hadoop/tmp
mkdir  /root/hadoop/var
mkdir  /root/hadoop/dfs
mkdir  /root/hadoop/dfs/name
mkdir  /root/hadoop/dfs/data

配置变量

>vi  /hadooplocation/etc/hadoop/hadoop-env.sh
export JAVA_HOME={$JAVA_HOME}

vi yarn-env.sh

#export JAVA_HOME=${JAVA_HOME}
去掉#，改为
export JAVA_HOME=/javapath
说明：修改为自己的JDK路径。此处可以与系统JAVA环境一致，也可以单独配置。

修改core-site.xml

vi core-site.xml
<configuration>
	<!-- 指定hdfs的nameservice为ns -->
    <!--单一namenode使用fs.default.name-->
    <!--否则使用fs.defaultFS-->
    <property>
　　　　<name>fs.default.name</name>
　　　　<value>hdfs://master:9000</value>
    </property>
	<property>
    <!--指定hadoop数据临时存放目录-->
　　　　<name>hadoop.tmp.dir</name>
　　　　<value>/root/hadoop/tmp</value>
　　</property>
</configuration>
